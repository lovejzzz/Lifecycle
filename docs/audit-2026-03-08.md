# Lifecycle Agent — Project Audit

Date: March 8, 2026

## Overview

Lifecycle is an agent-first visual workflow builder. Users converse with CID (two personalities: Rowan and Poirot) to generate, execute, and analyze node-based workflows on a React Flow canvas. The stack is Next.js 16, React 19, Zustand 5, Framer Motion, and Tailwind CSS 4.

This audit covers architecture, security, performance, code quality, error handling, state management, accessibility, dependencies, and testing.

## Findings

### 1. No Test Infrastructure [Critical]

No test files, no test framework, no test script. Pure functions like `topoSort`, `resolveOverlap`, `analyzeIntent`, and the LLM response normalization in `route.ts` are untested.

**Risk:** Any refactor (especially the store split) has no safety net.

**Recommendation:**
- Add Vitest + React Testing Library
- Priority targets: `graph.ts`, `intent.ts`, `route.ts` normalization logic
- Add `npm test` script and include in `npm run check`

### 2. Monolithic Store (4,488 lines) [High]

`src/store/useStore.ts` contains state shape, node/edge CRUD, execution logic, chat/LLM orchestration, workflow analysis, agent personality logic, import/export, and compression — all in one file.

**Risk:** Hard to reason about, hard to test, encourages tight coupling.

**Recommendation:** Split into focused modules:
- `store/state.ts` — state shape + persistence
- `store/nodes.ts` — node/edge CRUD
- `store/execution.ts` — executeWorkflow, executeNode
- `store/chat.ts` — chatWithCID, generateNodeContent
- `store/analysis.ts` — healthScore, validate, criticalPath

Zustand supports slices pattern natively.

### 3. Accessibility Gaps [High]

Multiple WCAG 2.1 AA violations:

- **No ARIA labels:** Icon buttons in Canvas (duplicate, lock, delete, ask CID) have no accessible names. Chat input, mode toggle, search input — all unlabeled.
- **No keyboard navigation:** Context menu (right-click only), legend/shortcuts panels, autocomplete dropdown — none keyboard-navigable. No focus trap in modals.
- **Color-only status:** Node status indicated solely by border color (green/yellow/red). Unusable for color-blind users.
- **Missing form labels:** NodeDetailPanel inputs use visual text but no `<label htmlFor>` association.

**Recommendation:** Add `aria-label` to all icon buttons, `role="menu"` to context menus, focus trapping to modals, and icon+text indicators alongside color for status.

### 4. Input Validation Gaps [High]

- **No length limits on LLM output:** `route.ts` normalizes categories/labels but doesn't cap string length. A malicious or hallucinating model could return 1MB node labels.
- **No size limit on workflow import:** `importWorkflow` parses JSON without checking payload size first — could freeze the browser on large inputs.
- **No client-side rate limiting:** Users can spam chat requests with no cooldown.

**Recommendation:**
- Add max lengths: label (200 chars), description (5,000), content (50,000)
- Add import size check: reject payloads > 5MB before parsing
- Add 1-second cooldown between chat submissions

### 5. Swallowed Errors and Missing User Feedback [Medium]

Several catch blocks silently discard errors:
- `localStorage.setItem` failure — no log, no recovery
- `autoDescribe` AI failure — silent fallback, no toast
- `chatWithCID` timeout (45s) — AbortSignal fires but no user-facing "request timed out" message

Partial state after failure: if `generateNodeContent` fails mid-stream, nodes are partially created with no rollback.

**Recommendation:**
- Log all caught errors (at minimum `console.warn`)
- Show toast on timeout: "Request timed out — try again"
- Use `pushHistory()` before multi-node operations so failures can undo cleanly

### 6. Performance Bottlenecks [Medium]

- **Drag-time overlap resolution:** `onNodeDrag` calls `resolveOverlap` on every pixel of movement — re-serializes the entire node list per frame. Should debounce or use requestAnimationFrame.
- **Unbounded message list:** CIDPanel renders all messages as DOM nodes. At 1,000+ messages, scrolling degrades. Should virtualize or lazy-load.
- **Over-broad selectors:** Canvas subscribes to 15+ store properties — any change re-renders the entire canvas. Should split into focused selectors.
- **Layout recalc after every node add:** `postBuildFinalize` calls `optimizeLayout` after each node. When AI generates 50 nodes in a batch, this fires 50 times. Should batch.
- **Serialization cost:** `JSON.stringify(entireState)` on every mutation for localStorage persistence. With large workflows this is 10MB+ per write.

### 7. Code Complexity and Constants [Medium]

- `buildNodesFromPrompt` in `intent.ts` is 261 lines of linear if/else logic. Should decompose into per-category builder functions.
- Magic numbers scattered: `NODE_W=280`, `NODE_H=160`, `MAX_HISTORY=30`, chained timeouts in `postBuildFinalize` (400, 1200, 2000, 2500ms) — none documented.
- Edge label constants defined in three places: `Canvas.tsx`, `route.ts` (KNOWN_LABELS), `graph.ts` (ANIMATED_LABELS). Should have one source of truth.
- Dead code: `buildMemory` populated in `recordBuild` but never read. `lastExecutionSnapshot` only used in command-invoked `diffLastRun`.

### 8. Missing LifecycleNode Memoization [Medium]

`LifecycleNode` (the custom React Flow node component) is not wrapped in `React.memo()`. Every store change that touches `nodes` re-renders every node on the canvas. With 100+ nodes this is noticeable.

**Recommendation:** Wrap in `React.memo` with a shallow equality check on `data` props.

### 9. No API Client Abstraction [Low]

Five inline `fetch('/api/cid', ...)` calls in the store, each constructing headers and body independently. Duplicated error handling, duplicated `taskType` logic.

**Recommendation:** Extract a `cidClient.ts` module:
```typescript
export async function callCID(opts: { systemPrompt: string; messages: Message[]; model?: string; taskType?: TaskType }): Promise<CIDResponse>
```

### 10. Loose Version Pins [Low]

`package.json` uses `^` for all dependencies (e.g., `"next": "^16.1.6"`). The `^` range allows minor+patch updates, which is fine for development but risky for reproducible deploys.

**Recommendation:** Use a lockfile (`package-lock.json`) and consider `~` pins for production stability.

## What's Working Well

- **Server-side API routing:** All LLM traffic goes through `/api/cid` — no browser-side secrets.
- **Provider abstraction:** Server resolves model → provider (Anthropic, DeepSeek, OpenRouter) with clean fallback chain.
- **Task-aware temperature:** Creative generation gets 0.8, analysis gets 0.4 — smart differentiation.
- **Execution guardrails:** Trigger/dependency nodes passthrough without wasting API calls.
- **Agent personality system:** Rowan (action-first) and Poirot (investigative) have genuinely different behaviors, not just different welcome messages.
- **LLM response normalization:** Robust handling of non-standard model output — category mapping, edge label inference, ID resolution, markdown stripping.
- **Ephemeral messages:** Welcome-back greetings don't pollute persisted history.
- **Type system:** Full TypeScript strict mode. 13 node categories with color + icon mapping.
- **Graph algorithms:** topoSort, overlap resolution, free position finding — correct implementations.
