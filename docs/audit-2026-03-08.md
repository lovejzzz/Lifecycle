## What's Working Well

1. **Agent Intelligence**: 5-layer personality system (Temperament → Drive → Habit → Generation → Reflection) genuinely evolves agent behavior over time. Domain expertise tracking with decay is elegant.
2. **Eval Harness**: 60-test rotating pool covering 12+ industries, 4 task types, edge cases. Model override (`deepseek-chat` / `deepseek-reasoner`) enables cross-model comparison. 5 consecutive 100% scores.
3. **API Route Robustness**: Non-CID JSON detection, node reordering (output always last), multi-provider support (DeepSeek, Anthropic, OpenRouter), retry on 429, timeout handling.
4. **Graph Utilities**: Clean extraction in Round 68. topoSort, edge inference, overlap resolution all well-bounded.
5. **Dual Agent Personalities**: Rowan (Soldier) and Poirot (Detective) produce consistently distinct, character-appropriate output across models.

## Key Findings

### Security
- `src/app/api/cid/route.ts`: No rate limiting — vulnerable to API abuse (HIGH)
- `src/lib/graph.ts:158-176`: `markdownToHTML` escapes `<>` but doesn't use DOMPurify — XSS risk if LLM output is malicious (HIGH)
- `src/lib/types.ts:228`: Deprecated `apiKey` field still in NodeData interface — should be removed (LOW)

### Performance
- `src/store/useStore.ts`: 4,622 lines, 70+ methods — every action triggers subscriber re-renders (MEDIUM)
- `src/lib/graph.ts:100-130`: `resolveOverlap` called on every drag without debouncing (LOW)
- `src/lib/graph.ts:73-98`: `findFreePosition` tries 50 collision attempts — acceptable for <20 nodes, degrades for dense graphs (LOW)

### Code Quality
- Lint warnings: 23 → 9 across two rounds (fixed 14 total: unused imports, variables, args in TopBar, reflection, agents, useStore)
- `src/lib/reflection.ts:557-565`: `processAllReflections` and `detectPatterns` are stubs — legacy compat (LOW)
- `src/lib/types.ts:108,170`: `HabitPattern` and `ReflectionEntry` V1 interfaces unused — replaced by V2 (LOW)

### Error Handling
- No React error boundary at Canvas or CID panel level — unhandled errors crash the entire app (MEDIUM)
- `src/store/useStore.ts:959,2365`: Errors caught as `_err` but no user notification (MEDIUM)
- Edge label normalization silently maps unknown labels to 'drives' (LOW)

### Accessibility
- Icon-only buttons (undo, redo, export, lock) lack `aria-label` attributes throughout `TopBar.tsx` and `NodeDetailPanel.tsx` (MEDIUM)
- Dropdown menus lack `aria-expanded`, `aria-haspopup`, and focus trap (MEDIUM)
- Status changes animated with Framer Motion but no `aria-live` announcements (LOW)
- Section headers use `<span>` instead of semantic heading elements (LOW)

## Priority Actions

1. **Add rate limiting** to `/api/cid` — IP-based, 10-20 req/min per client
2. **Add React error boundaries** around Canvas and CID panel
3. **Add ARIA labels** to all icon-only buttons (quick accessibility win)
4. **Split useStore.ts** into domain-specific sub-stores (graph, CID, UI)
5. **Add unit tests** for `topoSort`, `inferEdgeLabel`, `analyzeIntent` — pure functions, easy to test
6. **Remove deprecated `apiKey`** field from NodeData type
